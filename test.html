Style Guide

Posted on Dec 7, 2011 under Guide

Hail2u.netで使われているマークアップやスタイルの解説兼プレビューのページです。マークアップはHTMLのソースを、スタイルについては全ての変更履歴を含むGitリポジトリかこのページで参照されている非圧縮のCSSファイルを参照してください。このページに限ってはマークアップにおかしいところが多々ありますが、その多くはプレビューを作る上でのやむを得ない事情によるものです。

バージョニング

フォーマットはSemantic Versioningを使っています。ただしAPIの変更などというものはあまりないので、以下のような条件でバージョン番号の増加を行っています:

  1. メジャー・バージョンは、レイアウトの大幅な置き換えやテーマカラーの変更など、一見して大きな違いを見分けられる変更を行った場合に増加されます。
  2. マイナー・バージョンは、新たなウィジェットの追加やフォントやカラム幅のような全体に影響を与えるものの変更など、既存の見た目を継承するもののユーザーに影響を与えるような変更を行った場合に増加されます。マークアップの変更への対応が必要だったりする場合もマイナー・バージョンが増加されます。
  3. パッチ・バージョンは、単なる修正や調整など、特にユーザーに影響を与えない変更に留まる場合に増加されます。スタイルの変更のためにマークアップの変更を余儀なくされる場合はパッチ・バージョンが増加されます。

デバッグ

レイアウトの位置確認のために、空いているところをダブルクリックするとグリッドをトグルすることができます。このグリッドはCSSグラデーションを利用した簡単なものなので、描画領域の幅によっては数ピクセル程ずれることがあります。

また、グローバル・ヘッダーのロゴをクリックすると記事個別ページでの特別な表示の仕方に切り替わります。

印刷向けスタイルシート

以前は別に書いていましたが、HTTPリクエストの削減のためにインライン化しました。フォントはセリフ系を採用しています。他はなるべく白黒になるように調節しているだけです。

コーディング上の決め事

以下はマークアップやスタイル、スクリプトにおけるコーディング規約もどきです。

マークアップ

  • HTML5のHTML文法を使ってUTF-8で書きます
  • インデントは半角空白を二文字使います
  • 要素名や属性名は全て小文字に統一します
  • 属性値を括るクオートは省略しません
  • 真偽値を取る属性値はその値を省略します
  • クラス名やIDはハイフンつなぎで指定します
  • JavaScript等で動的に生成されるコードも上記決め事に従います
  • Validator.nuでエラーが出ないようにします

スタイル

  • SassのSCSS記法を使ってUTF-8で書きます
  • ASCII範囲外の文字はエスケープします
  • インデントは半角空白を二文字使います
  • セレクターは一行につき一つに制限します
  • プロパティーも一行につき一つに制限します
  • 省略系のプロパティーはなるべく使いません
    • marginやpadding、 borderに限って使うことがあります
  • 値はなるべく一行(80文字)に収まるように書きます
    • linear-gradient()等の複雑なものに限って行を分けて書くことがあります
  • ベンダー拡張プリフィックス付きのプロパティーはその時点での実装を考慮して記述します
  • CSS Lintでチェックだけします
  • Sass
    • BEMの思想を流用し、ファイルを作成してブロックを、ネストでエレメントを、&を使ったネストでモディファイアを表現します。
    • 変数は$type_block__element--modifierという規則で命名します

スクリプト

  • JavaScriptをUTF-8で書きます
  • ASCII範囲外の文字はエスケープします
  • インデントは半角空白を二文字使います
  • JSHintでエラーが出ないようにします

ベース

ブラウザーのデフォルト・スタイルシートの差異の吸収のためにNormalize.cssを利用しています。ただし必要な物だけを取り込めるようにルールセットごとに分割しSassのパーシャルにしたものを使っています。

行あたりの文字数

Lorem ipsum dolor sit amet, consectetur adip*iscing elit. Suspendisse moll*is elementum leo sed feugiat.

直前の段落のアスタリスクは45文字目と75文字目に打ってあります。一行の文字数は45–75文字(line-height: 1.5の場合)と言われているので、このアスタリスクが行をまたいでいると適切な文字数で折り返されているということになります。このウェブサイトでは行間が1.618と少し広いことと、Windowsのブラウザーで使われることが多いメイリオが幅が広いことを基準にし、広めに設定しています。

視認性のために、対応している場合は構文強調を行っています。構文強調にはgoogle-code-prettifyを使用しています。

/**
 * nth element in the fibonacci series.
 * @param n >= 0
 * @return the nth element, >= 0.
 */
function fib (n) {
  var a = 1,
    b = 1;
    tmp;

  while (--n >= 0) {
    tmp = a;
    a += b;
    b = tmp;
  }

  return a;
}

console.log(fib(10));

語・句・文単位での意味付け

段落内では様々な意味づけを行うマークアップを使用しています。主に使うものはハイパーリンクを始め、強調やより強い強調であったり、出典元の明示やインラインでの引用、WWW等の略語、<code>等のコード断片、Enterなどのキー入力、単なるマークなど、あまり逸脱しない形でスタイルしています。

ただし、ハイパーリンクは段落内でのみ識別を容易にするため下線を表示させています。また、強調においては日本には斜体の文化は無いので太字としています。

編集

より良い編集履歴を残すため、インラインで追加した部分は特に何もせず、削除した部分は取り消し線を加えています。単なる追記等の場合はそのことを明示するだけに留め、insでマークアップはしていません。

表

図と同じように本文と明確に区別ができるように、下線を付けるようにしています。ヘッダーのみ下線は少し太くなります。

ブログ記事

Posted on Dec 8, 2013 under Miscellaneous

ブログでは見出しに続けてfooter要素が配置され、更新日時とカテゴリをそこで示します。記事を構成するそれぞれの要素にはMicrodataとMicroformatsを利用しており、容易に機械的に解釈できるようにしています。

<article
  itemprop="blogPost"
  itemscope
  itemtype="http://schema.org/BlogPosting"
  itemref="author">
  <h1 itemprop="name">...</h1>

  <footer>
    <p>Posted
      on
      <time
        datetime="..."
        itemprop="datePublished">M d, yyyy</time>
      under
      <a
        class="tag">
        href="..."
        <span itemprop="keywords">...</span>
      </a>
    </p>
  </footer>

  <div itemprop="articleBody">...</div>
</article>
