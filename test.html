It’s a living style guide of HTML, CSS & JavaScript for this Hail2u.net website.

Style Guide v6.75.2

Dec 7, 2011 · Guide

Hail2u.netで使われているマークアップやスタイルの解説兼プレビューのページです。マークアップはHTMLのソースを、スタイルについては全ての変更履歴を含むGitリポジトリかこのページで参照されている非圧縮のCSSファイルを参照してください。このページに限ってはマークアップにおかしいところが多々ありますが、その多くはプレビューを作る上でのやむを得ない事情によるものです。

デバッグ

レイアウトの位置確認のために、空いているところをダブルクリックするとグリッドと要素のアウトラインをトグルすることができます。グリッドはCSSグラデーションを利用した簡単なもののため、描画領域の幅によっては数ピクセル程ずれることがあります。

また、このセクションの大見出しをクリックすることでタグラインの切り替えを、日付をクリックすることでアイキャッチ画像の切り替えを、それぞれ行うことができます。

印刷向けスタイルシート

フォントはセリフ系を採用しています。他はなるべく白黒になるように調節しているだけです。

コーディング上の決め事

It’s just a rule. Nothing can stop me now, ’cause I don’t care.

以下はマークアップやスタイル、スクリプトにおけるコーディング規約もどきです。

マークアップ

  • HTML StandardのHTML文法を使ってUTF-8で書きます
  • インデントは半角空白を二文字使います
  • 要素名や属性名は全て小文字に統一します
  • 属性値を括るクオートは省略しません
  • 真偽値を取る属性値はその値を省略します
  • 属性の記述順
    1. id属性
    2. class属性
    3. style属性
    4. その他属性
    5. WAI-ARIA (role属性とaria-*属性)
    6. カスタムデータ属性(data-*属性)
    7. Microdata (item*属性)
  • クラス名やIDはハイフンつなぎで指定します
  • JavaScript等で動的に生成されるコードも上記決め事に従います
  • Validator.nuでエラーが出ないようにします

スタイル

  • SassのSCSS記法を使ってUTF-8で書きます
  • インデントは半角空白を二文字使います
  • セレクターは一行につき一つに制限します
  • 定義は一行につき一つに制限します
  • プロパティーのアルファベット順に定義を記述します
  • 省略系のプロパティーはなるべく使いません
    • marginやpadding、 borderプロパティーに限って使うことがあります
  • 値はなるべく一行(80文字)に収まるように書きます
    • linear-gradient()等の複雑なものに限って行を分けて書くことがあります
  • ベンダー拡張プリフィックス付きのプロパティーはその時点での実装を考慮して記述します
  • Sass
    • BEMの思想を流用し、ファイルを作成してブロックを、ネストでエレメントを、&を使ったネストでモディファイアーを表現します
  • SCSS-Lintでエラーが出ないようにします

スクリプト

  • JavaScriptをUTF-8で書きます
  • インデントは半角空白を二文字使います
  • ESLintでエラーが出ないようにします

ベース

ブラウザーのデフォルト・スタイルシートの差異の吸収のためにNormalize.cssを利用しています。ただし必要な物だけを取り込めるようにルールセットごとに分割しSassのパーシャルにしたものを使っています。

アイコン

ロゴとほぼ同じ画像を正方形にしたSVGフォーマットの画像から生成されています。PNGフォーマットへ変換したものや、更にそのPNGフォーマットの画像をアイコン形式にまとめたものがブラウザーによって適宜使われます。

レイアウト

コンテンツのおさまり具合によってレイアウトが切り替わります。ブラウザーの大きさを変更したりすると、その様子が確認できるでしょう。以下のスクリーンショットは切り替わるそれぞれのレイアウトを収めたものですが、実際にはフォントサイズも変化するのでもう少し複雑に変化します。

行あたりの文字数

Lorem ipsum dolor sit amet, consectetur adip*iscing elit. Suspendisse moll*is elementum leo sed feugiat.

直前の段落のアスタリスクは45文字目と75文字目に打ってあります。一行の文字数は45–75文字(line-height: 1.5の場合)と言われているので、このアスタリスクが行をまたいでいると適切な文字数で折り返されているということになります。このウェブサイトでは行間が1.618と少し広いことと、Windowsのブラウザーで使われることが多いメイリオが幅が広いことを基準にし、広めに設定しています。

セクション

HTML5が定義するところのセクションを構成する要素についてです。

body

文字はブラウザーの初期設定のまま(通常は16px)で表示されます。行間はx-heightという概念のない日本語がメインなので少し多めに1.618としています。狭い画面でコンテンツに適切な幅が確保できない場合は1.456へ落とし、おさまり具合を調節しています。

セクション構成物

セクションを作成する、section、nav、articleとaside要素では、特に何もスタイルを指定していません。ただし予期せぬ回り込み等を回避するためにclearfixだけは指定することがあります。

見出し

セクションの見出しはh1要素でマークアップされます。

<h1>...</h1>

小見出し

セクション内の小見出しはセクションの見出しであるh1要素に合わせてそのレベルを調節しています。多くの場合はh2要素のみが使われることでしょう。

<h2>...</h2>

コンテンツのグループ化

コンテンツで使われる基本的な要素についてです。

段落

様々な所で使用されるので、他とバッティングしないように相殺可能なマージン以外のスタイルは指定していません。

<p>...</p>

水平線

主に暗黙のセクションの区切りが挿入される所に恣意的に使用しています。線の代わりに表示されるアステリズムはSVGです。

<hr>

引用

内部で様々なスタイルが使用されるので、ブロックの左端に縦線を引く以外のスタイルは指定しません。字体に斜体を採用すべきところですが、メイリオが和文部分に斜体を持っていないのでそれを考慮して無指定にしています。マークアップは通常、全体をfigure要素で括り、引用元の表記はfigcaptionとcite要素を組み合わせて行います。

<figure>
  <blockquote>
    <p>...</p>
  </blockquote>

  <figcaption>— <cite>...</cite></figcaption>
</figure>

図

通常はfigure要素を使っています。既にblockquote要素で使われているように内容は画像とは限らないので、強いスタイルは指定していません。

<figure>
  <img alt="..." src="...">
  <figcaption>...</figcaption>
</figure>

画像に続けて書いたfigcaption要素は、広い画面では図全体と横に並びます。

語・句・文単位での意味付け

段落内では様々な意味づけを行うマークアップを使用しています。主に使うものはハイパーリンクを始め、強調やより強い強調であったり、出典元の明示やインラインでの引用、WWW等の略語、<code>やvariable等のコード断片、Enterなどのキー入力、単なるマークなど、あまり逸脱しない形でスタイルしています。

ただし、ハイパーリンクは段落内でのみ識別を容易にするため下線を表示させています。また、強調においては日本には斜体の文化は無いので太字としています。

編集

より良い編集履歴を残すため、インラインで追加した部分は特に何もせず、削除した部分は取り消し線を加えています。単なる追記等の場合はそのことを明示するだけに留め、ins要素でマークアップはしていません。

共有ボタン

ウェブログの記事には、その最後にいくつかのソーシャル・ネットワークで共有するためのリンクが追加されます。現在はTwitter、Facebook、そしてはてなブックマークの3種類です。

各ソーシャル・ネットワークが提供する公式の共有ボタンやそれに類するものはできる限り使用しません。主にデザインの一貫性を確保することが難しいからですが、その役割と利用頻度に比べて、JavaScriptライブラリーの読み込みコストが高いということもあります。
